# IP Address Formatting

`ip_address_formatting` — это утилитарный скрипт для нормализации и разворачивания IP-адресов из “грязного” текстового ввода в явный список корректных адресов.

Скрипт предназначен для ситуаций, когда IP-адреса приходят в сокращённой, смешанной или частично размеченной форме: с диапазонами, перечислениями последних октетов, дополнительными подписями, комментариями и нестандартными разделителями. На выходе он преобразует такие записи в нормализованный набор отдельных IP-адресов, пригодных для дальнейшей обработки.

## Что умеет

- раскрывает сокращённые записи IP-адресов;
- обрабатывает диапазоны последних октетов;
- поддерживает перечисления через `/`;
- работает со смешанными форматами в одной строке;
- игнорирует лишние текстовые вставки, подписи и служебные пометки;
- подготавливает адреса для последующей передачи в другой рабочий процесс.

## Примеры входных форматов

Скрипт рассчитан на обработку записей вида:

```

        "80.80.141.184/185", "80.80.141.37 - clu; 80.80.141.38/39",
        "10.132.138.154/155", "10.201.93.157/158",
        "10.76.64.253/254", "10.150.141.92-95/31-32",
        "134.150.141.134-137", "134.132.167.142/143, 65.134.65.11/12",
        "10.31.31.165/166/167", "10.36.141.139/140/141",
        "13.10.68.25/32", "ctl1: 13.10.68.36; ctl2: 13.10.68.38", 
        "10.1.6.34-clu; 10.138.92.41/42",
        "10.132.43.33 - clu; 10.55.77.34/35", "10.88.1.6-9",
        "10.88.1.43/44/55/66/77/88/99-110",
        "134.150.141.92-95/31-32/40-46",
```

## Основная идея

На вход подаётся строка или список строк с IP-адресами в неполном или нестандартизированном виде.  
Скрипт извлекает адресную основу, раскрывает диапазоны и перечисления, очищает служебные символы и возвращает итоговый список отдельных IP-адресов в нормальном формате.

## Где полезен

Скрипт удобен в прикладных задачах, где IP-адреса приходится собирать вручную из разных источников: переписок, таблиц, конфигов, заявок, комментариев или внутренних описаний инфраструктуры.

## Ограничения

- утилита ориентирована на практические шаблоны, встречавшиеся в реальных данных;
- поддержка новых нестандартных форматов может требовать доработки;
- основной фокус сделан на рабочую полезность, а не на универсальный парсер всех возможных вариантов записи IP.

## Статус

Проект решает прикладную задачу нормализации IP-адресов и может служить базой для дальнейшего расширения и формализации логики разбора.

# Nmap Async

`nmap_async` — это утилитарный скрипт для массового асинхронного сканирования IP-адресов с помощью Nmap в многопоточном режиме.

Скрипт рассчитан на обработку больших списков адресов и предназначен для случаев, когда нужно быстро прогнать множество хостов с одинаковым набором параметров сканирования, сохранив результат по каждому IP в отдельный XML-файл.

## Основные возможности

- многопоточная обработка большого списка IP-адресов;
- запуск Nmap с заранее заданными параметрами сканирования;
- сохранение результата по каждому адресу в отдельный XML-файл;
- автоматическое формирование текстового файла со списком созданных файлов;
- фиксация общего времени выполнения;
- возможность вручную менять:
  - количество потоков;
  - список IP-адресов;
  - параметры сканирования.

## Как работает

1. Внутрь скрипта передаётся список IP-адресов.
2. Скрипт распределяет адреса между потоками.
3. Для каждого IP запускается отдельное Nmap-сканирование.
4. Результат сохраняется в XML-файл с уникальным именем.
5. После завершения работы дополнительно создаётся `names_files.txt`, где:
   - перечислены все созданные файлы;
   - указано общее время выполнения.

## Практическое назначение

Скрипт полезен для массового технического опроса инфраструктуры, когда нужно быстро собрать результаты по большому числу адресов и получить их в удобном для дальнейшей обработки формате.

## Особенности использования

- запускать скрипт лучше в отдельной директории, так как он создаёт большое количество XML-файлов;
- объём выходных данных напрямую зависит от количества IP-адресов;
- производительность зависит от числа потоков, сетевой доступности хостов и параметров сканирования;
- настройки потоков и параметры Nmap задаются внутри скрипта и могут быть адаптированы под конкретную задачу.

## Пример сценария

Скрипт подходит для пакетной обработки крупных списков адресов — от небольших выборок до масштабных наборов на десятки и сотни тысяч IP, если требуется автоматизировать однотипное сетевое сканирование и сохранить результаты в структурированном виде.

## Статус проекта

Проект представляет собой прикладной автоматизационный инструмент для массового Nmap-сканирования и может быть расширен в сторону более гибкой настройки, логирования и управления результатами.

# Date Changer

`Date Changer` — это консольная утилита для быстрого обновления диапазонов дат в текстовых скриптах и конфигурациях.

Скрипт автоматически находит поддерживаемые шаблоны дат, позволяет сдвигать диапазон вперёд относительно текущей даты в тексте, сбрасывать даты в инициализирующий период и актуализировать верхнюю границу до сегодняшнего дня. После обработки результат копируется в буфер обмена, а программа переключает окно через `Alt+Tab`, чтобы ускорить дальнейшую вставку и работу.

## Основные возможности

- сброс диапазона дат к инициализирующему периоду;
- сдвиг диапазона вперёд на указанное количество:
  - дней;
  - месяцев;
  - лет;
- обновление максимальной даты до текущей системной даты;
- автоматическое копирование результата в буфер обмена;
- автоматическое переключение на предыдущее окно после обработки;
- циклический режим работы для многократного использования без перезапуска.

## Поддерживаемые режимы

После вставки скрипта программа ожидает параметр в одном из форматов:

- `15d` — сдвиг на 15 дней;
- `2w` — сдвиг на 2 месяца;
- `1y` — сдвиг на 1 год;
- `l` — установить максимальную дату на текущую;
- `i` — сбросить диапазон на инициализирующий период:
  - `1899-01-01T00:00:00`
  - `1899-01-02T00:00:00`

Для выхода из программы:
- введите `exit`;
- либо закройте окно консоли вручную.

## Как это работает

1. Пользователь запускает `.bat`-файл рядом со скриптом.
2. В консоль вставляется текст скрипта или конфигурации.
3. Программа определяет поддерживаемый формат и извлекает даты.
4. Пользователь задаёт режим изменения диапазона.
5. Скрипт формирует обновлённый текст.
6. Результат автоматически:
   - копируется в буфер обмена;
   - переключает фокус на предыдущее окно через `Alt+Tab`.

## Поддерживаемые шаблоны

На текущем этапе утилита обрабатывает несколько заранее известных форматов, в том числе сценарии с полями вида:

- `dataset_max_date_to`
- `wf_dataset_max_date_to`
- `left_bound`
- `right_bound`
- SQL-подобные конструкции с `to_date(...)`

Если формат не распознаётся, программа сообщает об этом и предлагает обработать случай вручную.

## Ограничения

- дата не может быть больше текущей системной даты;
- проект пока ориентирован на запуск через `.bat` и консольный сценарий использования;
- некоторые нестандартные форматы могут не распознаваться;
- в отдельных случаях возможны ошибки при обработке редких или новых шаблонов.
